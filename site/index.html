Setup · one time

Get on the network

Your boxes live behind a private Tailscale network. You only have to do this once per machine.

1

Install Tailscale

Grab the app from the Mac App Store. Then open a terminal and check that the CLI works — running tailscale --help should print something.

Terminal says tailscale: command not found?

The app doesn't always add the CLI to your PATH. Paste this once to create a small wrapper that points at the app:

$mkdir -p /usr/local/bin
printf '#!/bin/sh\nexec /Applications/Tailscale.app/Contents/MacOS/Tailscale "$@"\n' > /usr/local/bin/tailscale
chmod +x /usr/local/bin/tailscale
2

Join the network

Connect your machine to the VPN. A browser window opens for you to sign in.

$sudo tailscale up --login-server=https://headscale.nixos.asia --hostname $(hostname -s)
Everyday use

Create a box & jump in

This is the whole workflow. Pick a name, and you're inside.

1

Create it

Pick any name for your box. This builds a fresh Linux container with a ready-to-go dev environment inside.

The first time you run it — and roughly once a week after — you'll be asked to sign in: the terminal prints a link and a short code. Open google.com/device, enter the code, and sign in with your Juspay Google account. That's what grants your SSH access — there are no keys to manage.

$nix run github:juspay/xyne-boxes create <name>
2

Jump in

That's it — this drops you into a shell on the box as the toor user.

$nix run github:juspay/xyne-boxes connect <name>
Prefer to just type ssh <name>?

Do this once and plain ssh <name> will work for every box you create.

1 · Let SSH pick up each box's config

$mkdir -p ~/.ssh
grep -qxF 'Include ~/.pu-state/*/ssh_config' ~/.ssh/config 2>/dev/null \
  || printf '\nInclude ~/.pu-state/*/ssh_config\n' >> ~/.ssh/config

2 · Forward your key so you don't copy it onto the box

$eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

Now just run ssh <name> to connect.

Inside the box

What you get

Every box is an Incus container cloned from a loaded base image — not a bare machine.

A real dev environment

git, vim, tmux, htop, direnv, starship and more, ready to go. VS Code Remote and nix-ld work out of the box.

Nix, already warm

Flakes are on and the box shares the host's /nix/store, so nix develop is fast and common Juspay dev-envs are pre-cached.

Yours, and it sticks around

You land as toor with your own home. It survives stop/start — nothing expires on its own. It's gone only when you run destroy.

More when you need it

list shows your boxes, fork <source> <name> clones one to branch an environment, and destroy <name> removes it.

Good to know

When your access expires

Your access lasts about a week. When it lapses, reconnecting fails with an error like this:

pu@pu: Permission denied (publickey,keyboard-interactive).
Connection closed by UNKNOWN port 65535

Nothing's broken — just run connect again and sign in the same way you did the first time (the link-and-code Google flow). About 20 seconds, and you're good for another week.

$nix run github:juspay/xyne-boxes connect

Seeing a GitHub 403 error? Turn the VPN off and try again. If it keeps happening, set up a GitHub token for a higher rate limit.
