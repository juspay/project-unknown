§ 01 Remote dev boxes over Tailscale

A remote box,
one SSH away.

xyne-boxes spins up a fresh development box and drops you straight into it. One command creates the box, Tailscale puts you on the network, and short-lived SSH certificates handle the login — no keys to copy, nothing to configure by hand.

  • Createnix run github:juspay/xyne-boxes create
  • Networkheadscale.nixos.asia
  • LoginGoogle SSO · 1-week certs
§ 02

Get on
the network.

Prerequisites

1. Install Tailscale — grab it from the Mac App Store. Make sure the CLI works: tailscale --help should run in your terminal. If it doesn't, create the CLI wrapper:

tailscale · cli wrapper
$mkdir -p /usr/local/bin && printf '#!/bin/sh\nexec /Applications/Tailscale.app/Contents/MacOS/Tailscale "$@"\n' > /usr/local/bin/tailscale && chmod +x /usr/local/bin/tailscale

2. Add your host machine to the VPN

tailscale · up
$sudo tailscale up --login-server=https://headscale.nixos.asia --hostname $(hostname -s)
§ 03

Create
& access.

Workflow

I.

Create the box

Run the command below. On first run you'll authenticate via Google — the same SSO flow as onboarding.

$nix run github:juspay/xyne-boxes create <name>
II.

Set up ssh-agent

Recommended

Forward your key with ssh-agent so you never copy a private key onto the box.

$eval `ssh-agent`
ssh-add <path-to-your-private-key>
III.

Enable ssh <name>

Wire the generated per-box config into your SSH config once. It's idempotent — safe to re-run.

$grep -qxF 'Include ~/.pu-state/*/ssh_config' ~/.ssh/config 2>/dev/null || { mkdir -p ~/.ssh && printf '\nInclude ~/.pu-state/*/ssh_config\n' >> ~/.ssh/config; }
IV.

Connect

You're in:

$ssh <name>

Don't care about using ssh directly? Skip steps II–III and connect through the client instead: nix run github:juspay/xyne-boxes connect <name>.

§ 04

Renew
your cert.

Troubleshooting

SSH certificates are valid for one week. When yours expires, a connection attempt fails like this:

pu@pu: Permission denied (publickey,keyboard-interactive). Connection closed by UNKNOWN port 65535

Renewing takes one command and a browser sign-in:

Step 1 · Run the connect command

Either connect or list triggers a fresh certificate:

nix run github:juspay/xyne-boxes connect

Step 2 · Authenticate via Google

The command prints a URL and a code:

  1. Visit google.com/device and enter the code shown in your terminal (XXX-XXX-XXXX).
  2. Sign in with your Juspay Google account — the same flow as onboarding.
Step 3 · SSH in as usual

Once signed in, your cert is renewed and ssh <name> works again.

⚠ Hit a GitHub 403 rate-limit error? Disconnect from the VPN and retry — that usually fixes it. For a higher limit, set up a GitHub PAT.
